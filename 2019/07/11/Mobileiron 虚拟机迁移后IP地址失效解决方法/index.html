Mobileiron 虚拟机迁移后IP地址失效解决方法

Veröffentlicht am 2019-07-11 | in Mobileiron | Kommentare: | Aufrufe:

Product:Mobileiron Core、Sentry
Version:All
Platform:Virtual System

Symptoms

在虚拟机平台对Mobileiron服务器克隆或迁移后，会发现克隆后的服务器启动后无法通过网络正常访问。

Cause

该问题是由于克隆或迁移后的虚拟机MAC地址已经自动更新，但服务器记录的网卡配置信息绑定了之前旧的MAC地址，这导致服务器会将该网卡应用新的网卡配置信息，默认为空的IP地址。进而出现了克隆后的服务器启动后无法通过网络正常访问的问题。

Solution

进入Linux SHELL

进入Linux SHELL的方法请咨询Mobileiron技术支持

修改网卡配置信息
  1. 使用ifconfig -a命令查看当前网卡配置信息，可以看到当前配置绑定的网卡是eth1

  2. 使用cd命令前往/etc/sysconfig/network-script目录下，可以看到目录中存在两个配置文件分别是ifcfg-eth0和ifcfg-eth1.
  3. 使用more命令分别查看两个配置文件信息，发现ifcfg-eth0包含旧的网卡配置信息，但ifcfg-eth1的配置信息则为默认设置。

  4. 使用vi编辑器对调两个网卡的配置信息，注意DEVICE参数保持不变。
  5. 接着使用service network restart命令重启网络即可
